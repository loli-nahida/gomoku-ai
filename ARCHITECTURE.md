# 🧠 五子棋 AI 架构原理

## 整体架构

```
玩家落子 → 游戏引擎 → AI 引擎 → MCTS 搜索 → 神经网络评估 → 返回最佳落子
```

参考 DeepMind **AlphaZero** 架构，简化适配浏览器端纯 JS 运行。

---

## 一、游戏引擎 (`game.js`)

15×15 棋盘，二维数组表示：`0=空, 1=黑, 2=白`

### 核心功能

- **落子合法性检查**
- **胜负判定**：沿4个方向（横、竖、两条对角线）检查5子连珠
- **威胁评估**：对每个空位计算落子后形成的棋型

```js
// 威胁评估
evaluateThreat(r, c, player) {
  // 检查4个方向，统计连子数和开放端
  // 活四(两端开放) → 1.0
  // 冲四(一端开放) → 0.5
  // 活三 → 0.3
  // ...
}
```

## 二、神经网络 (`neural-net.js`)

AlphaZero 风格双头网络，纯 JS 前向推理。

### 网络结构

```
输入层         → 15×15×6 特征平面
卷积层         → 3×3 Conv, 64 filters
残差块 ×4     → Conv→BN→ReLU→Conv→BN + 跳跃连接
┌────────────┬─────────────┐
│  策略头      │  价值头       │
│  Conv→FC    │  Conv→FC→FC  │
│  →225概率   │  →[-1,1]估值  │
│  (落子概率)  │  (局面判断)   │
└────────────┴─────────────┘
```

### 6个输入特征平面

| 通道 | 含义 |
|------|------|
| 0 | 我方棋子位置 |
| 1 | 对方棋子位置 |
| 2 | 最后一步位置 |
| 3 | 倒数第二步位置 |
| 4 | 每个空位的威胁程度 |
| 5 | 合法落子位置 |

### 关键参数

- 总参数量：4,469,218
- 激活函数：ReLU（隐藏层）、Softmax（策略头）、Tanh（价值头）
- 归一化：BatchNorm

## 三、蒙特卡洛树搜索 (`mcts.js`)

MCTS 是 AI 的"思考过程"，模拟未来对弈。

### 搜索树结构

```
        根节点（当前局面）
       /    |    \     ...
     落子A  落子B  落子C
      |      |      |
    局面A   局面B   局面C  ← 神经网络评估
     ...    ...    ...
   模拟到终局，回传胜负
```

### 四步循环

**1. 选择 (Selection)**

用 PUCT 公式选择最优子节点：

```
score = Q(s,a) + c × P(s,a) × √(N父) / (1 + N子)
        ↑实际胜率   ↑先验概率   ↑探索鼓励
```

**2. 扩展 (Expansion)**

到达叶节点后，用神经网络预测所有合法落子的概率，创建子节点。

**3. 评估 (Evaluation)**

用神经网络价值头评估当前局面 [-1, +1]。

**4. 回传 (Backpropagation)**

将评估值沿路径回传，更新每个节点的访问次数和价值统计。

### 最终决策

搜索完成后，**访问次数最多的子节点** = 最佳落子。

## 四、启发式评估（备选方案）

关闭神经网络时的纯规则评估：

## 五、自我对弈训练 (`train.js`)

```
┌─────────────────────────────────────┐
│          自我对弈循环                 │
│                                     │
│  1. 两个AI用MCTS对弈                 │
│  2. 记录每步：局面 + MCTS策略 + 结果  │
│  3. 胜方位置标记 value = +1          │
│     负方位置标记 value = -1          │
│  4. 收集为训练数据                    │
│  5. 重复                             │
└─────────────────────────────────────┘
```

### 训练数据

| 指标 | 数值 |
|------|------|
| 总训练局数 | 4,078 |
| 位置样本数 | ~27,000 |
| 黑胜率 | ~64% |
| 白胜率 | ~36% |
| 平均每局步数 | ~20 |

## 六、数据流

```
玩家点击棋盘
    ↓
游戏引擎验证落子合法性
    ↓
AI 引擎接收当前局面
    ↓
┌─ 入门模式：启发式评估（瞬间响应）
│
└─ 进阶+模式：
    MCTS 启动
    ├─ 每次模拟：选择→扩展→NN评估→回传
    ├─ 重复 N 次（100~5000次）
    └─ 返回访问次数最多的落子
    ↓
游戏引擎更新棋盘
    ↓
检查胜负 → 弹窗 / 等待玩家
```

## 七、关键设计决策

| 决策 | 原因 |
|------|------|
| 入门模式纯启发式 | Termux等低端设备跑不动MCTS+NN |
| 默认关闭NN | NN前向推理慢，入门模式不需要 |
| 每500局自动保存 | 防止OOM崩溃丢失训练数据 |
| Canvas绘制棋盘 | 比DOM操作性能好，支持触摸 |
| 同步AI（非Worker） | Termux WebView不支持Web Worker |
| 7档难度 | 满足不同设备性能和玩家水平 |
