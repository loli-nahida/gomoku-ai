# 🎓 五子棋 AI 训练原理（通俗版）

## 一句话概括

> 让 AI 自己跟自己下棋，下得多了，就变强了。

就像一个从来没下过棋的人，自己左手跟右手下，下了几千盘之后，慢慢摸索出规律。

---

## 第一步：自己跟自己下棋（自我对弈）

```
AI(黑) ←→ AI(白)
  ↓
每一步都记录：
  - 当前棋盘长什么样
  - AI 选了哪个位置（以及考虑过哪些位置）
  - 最后谁赢了
```

两个 AI 互相下，每下一步都拍照（记录棋盘状态），最后看谁赢了。

---

## 第二步：给每一步打标签

下完一盘之后，回头给每一步"打分"：

```
黑棋赢了：
  黑棋走的每一步 → 标记为 "好棋" (+1)
  白棋走的每一步 → 标记为 "坏棋" (-1)

白棋赢了：
  反过来

平局：
  所有步骤 → 标记为 "还行" (0)
```

就像复盘：赢了的人走的棋都是好棋，输了的人走的棋都是坏棋。

---

## 第三步：AI 怎么"思考"每一步（MCTS 搜索）

AI 不是随便选位置的，它会**在脑子里模拟未来的走势**：

```
当前局面
├── 如果我下这里 → 对手可能下那里 → 然后我再... → 赢了！
├── 如果我下那里 → 对手可能... → 输了...
└── 如果我下这里 → ... → 平局
```

这叫**蒙特卡洛树搜索（MCTS）**：

1. **随便试试**：从当前局面开始，随机往下走，直到分出胜负
2. **记住结果**：哪个位置试了之后赢的多，就多试几次
3. **反复模拟**：模拟几百几千次之后，哪个位置赢的概率最高，就选哪个

就像你站在岔路口，先每条路都走几遍试试，最后选走得最顺的那条。

---

## 第四步：神经网络——AI 的"直觉"

每次模拟都要判断"这个局面好不好"，这就是神经网络干的事。

### 网络结构（简化版）

```
输入：当前棋盘（15×15 的棋盘，6层信息）
  ↓
卷积层：提取局部特征（比如"这里有个活三"）
  ↓
残差块 ×4：反复提炼信息
  ↓
┌──────────┬──────────┐
│ 策略头    │ 价值头    │
│ "下哪好？" │ "赢面多大？"│
│ →225个概率 │ →[-1,+1]  │
└──────────┴──────────┘
```

- **策略头**：告诉 MCTS "先试试哪些位置"（缩小搜索范围）
- **价值头**：告诉 MCTS "这个局面好不好"（不用真的下完就能判断）

就像人类的"棋感"——看到一个局面，直觉就知道谁占优。

### 输入的6层信息

```
层0：我的棋子在哪
层1：对手的棋子在哪
层2：最后一步下在哪
层3：倒数第二步下在哪
层4：每个位置的威胁程度（能形成什么棋型）
层5：哪些位置可以下
```

---

## 第五步：怎么让网络变聪明（训练）

### 收集到的数据

```
每一步 = {
  棋盘状态,    ← 输入
  MCTS的落子概率,  ← 目标输出1（策略）
  最终胜负      ← 目标输出2（价值）
}
```

### 训练方式：随机试探法

不用复杂的反向传播，用最朴素的方法：

```
对于网络里的每个数字（权重）：

1. 先算一下当前的"错误程度"（loss）
2. 随机给这个数字加一点点噪声
3. 再算一次"错误程度"
4. 如果错误变小了 → 保留这个改动（方向对了）
   如果错误变大了 → 撤回（方向错了）
```

就像蒙着眼睛找下坡路：
- 试探着迈一步
- 如果感觉更低了，继续往那边走
- 如果感觉更高了，退回换方向

虽然慢，但确实能走到低谷。

---

## 第六步：完整循环

```
┌────────────────────────────────┐
│  1. 两个AI下棋（自我对弈）       │
│  2. 记录每一步的棋盘+选择+结果   │
│  3. 收集很多盘棋的数据           │
│  4. 用数据训练神经网络           │
│  5. 训练完的网络让AI更有"直觉"   │
│  6. 用更强的AI继续下棋           │
│  7. 循环...                     │
└────────────────────────────────┘
```

训练越多 → 直觉越准 → 搜索越高效 → AI 越强

---

## 为什么纯 JS 就能跑？

| 传统方法 | 本项目 |
|---------|--------|
| 需要 Python + PyTorch | 纯 JavaScript |
| 需要 GPU | 只用 CPU |
| 反向传播（计算梯度） | 随机扰动（试探法） |
| 框架自动求导 | 手写每一层的计算 |
| 模型几 GB | 模型 ~100MB |

关键优化：
- MCTS 用**启发式评估**（数棋型），不跑神经网络，单步 <1ms
- 神经网络用**纯数组运算**，没有框架开销
- 训练用**扰动法**，不需要写反向传播代码
- 数据用 **Float32Array**，内存紧凑

## 打个比方

整个过程就像培养一个棋手：

| 阶段 | 对应 | 说明 |
|------|------|------|
| 婴儿期 | 初始化 | 随机权重，啥也不会 |
| 自学 | 自我对弈 | 左手跟右手下，摸索规律 |
| 复盘 | 打标签 | 赢了的棋是好棋，输了的是坏棋 |
| 直觉 | 神经网络 | 看一眼棋盘就知道谁占优 |
| 思考 | MCTS搜索 | 在脑子里推演未来几步 |
| 训练 | 权重更新 | 根据经验调整"直觉" |
| 成长 | 循环 | 下得越多，直觉越准 |

训练 10000 局之后，这个"婴儿"就变成了一个会下棋的"棋手"。
